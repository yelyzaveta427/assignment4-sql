В порівнянні з неоптимізованим запитом (2246 мс), час виконання оптимізованого запиту виявився меншим (1564 мс) через те, що використано індекс на ід пасажира. 
В плані виконання оптимізованого запиту на 13,16 рядках видно, що використано сканування індексом, в той час як в плані неоптимізованого запиту на рядках 14,19,24 виконано послідвне
сканування, через що запит і виконувався повільніше.
